简历处理机器人开发，resume-robot
需求分析：
1. 面向企业人事。企业人事会收到很多简历，其中90%是不符合要求的简历，浪费大量的时间；
2. 预筛选。首先将收到的简历中，明显不符合要求的简历筛选掉；；
3. 将其他简历收纳进企业人才库，放进部门岗位文件夹；
4. 将简历解析，对人才进行画像；对岗位进行画像，将简历和岗位进行匹配；
5. 将简历进行打分，分层；对简历进行评价，并分析风险点；
6. 将优质的简历做成推荐报告（包括简历信息、综合评价、风险点），推荐给用人部门。

扩展功能设计：
目前已经具备较完善的简历解析和推荐报告生成功能。接下来，进行功能扩展，满足企业人事的简历处理需求。
1. 需要一个简历系统管理所有的简历；前期可以由hr上传，后期可以扩展到从邮箱、本地文件夹直接拉取；
2. 需要预筛选规则引擎：允许HR自定义“硬门槛”。对简历进行初筛。简历上传后，可执行初筛，过滤后的简历存到过滤箱。
3. 岗位管理系统。允许hr添加发布岗位，岗位发布后，对岗位进行解析画像，并对岗位解析信息进行存储，用于和候选人简历进行匹配。同时，岗位发布后，即生成岗位简历库容器，用于储存投递该岗位的简历。
4. hr可以批量上传简历到公司简历库，也可以直接上传到岗位简历库；
5. 上传到岗位简历库的简历，可执行预筛选进行初筛，被过滤的简历自动保存到过滤箱。
6. 对初筛后的简历，和岗位进行自动匹配，按照匹配度，生成标签：强烈推荐 / 推荐 / 谨慎推荐 / 不推荐。
7. 需要根据不同的岗位设计匹配模型。
8. 需要在后台设置公司信息，公司信息设置应该有利于LLM对候选人的简历进行匹配。公司信息应该是prompt的一部分或者一个层级。公司信息应该明确行业、产品、应用场景、公司文化等。如：公司是IC设计行业，主要做车规 / AI / 通信；对可靠性、安全的偏好
9. hr可在初筛后的简历中，选择简历进行批量生成推荐报告。

其他：
1. 支持手动上传 / 邮箱接收 / ATS 集成三种接入方式，适配企业级生产环境。
2. 第一阶段，实现MVP，支持手动上传。
3. 三类数据库协同：关系型数据库PostgreSQL ，文档数据库 或 对象存储MongoDB ，向量数据库Milvus 
4. 优先使用deepseek和千问大模型，豆包模型备选。
5. 公司信息、部门信息、岗位信息，可以起到prompt的作用，让大模型可以更好地理解岗位信息，理解企业需要招聘什么样的人，从而能更好地将简历和岗位进行匹配。
